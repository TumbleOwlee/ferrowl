# `ferrowl-ocpp`

The version-generic OCPP engine for `ferrowl`: Charging Station and CSMS roles over
JSON-on-WebSocket, supporting OCPP 1.6, 2.0.1, and 2.1.

The authoritative specification of this crate's behavior — the exhaustive per-version
action tables, the connection engine, error handling, and the TLS/auth security
profiles — lives in [`../docs/specs/ocpp/`](../docs/specs/ocpp/). Refer to that and
to the code rather than to a prose overview here.
